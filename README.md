# Claude Code Playbook

비개발자가 클로드 코드(Claude Code)를 배우는 단계별 학습 환경. 미니 프로젝트로 시작해서 풀 프로젝트로 성장한다.

---

## 왜 만들었나

매일 반복하는 업무를 자동화하고 싶지만, 코딩을 모르면 시작조차 어렵다. ChatGPT나 Claude 같은 챗봇은 "답"을 주지만, 실제로 파일을 만들고 작업을 대신 해주지는 않는다.

클로드 코드는 대화로 컴퓨터에게 직접 일을 시키는 CLI 도구다. 하지만 터미널, Git, CLAUDE.md 같은 개념이 처음인 사람에게는 진입 장벽이 높다.

이 플레이북은 그 장벽을 없애기 위해 만들었다. 코딩을 몰라도 한국어로 대화하면서, 작은 자동화부터 하나씩 만들어볼 수 있다.

---

## 어떻게 쓰나

### 시나리오 1: 처음 시작하는 사람

1. 설치 가이드(아래 "시작하기" 섹션)를 따라 클로드 코드를 설치한다.
2. 터미널에서 `claude` 를 실행하면, 클로드가 먼저 이름을 물어보고 대화를 시작한다.
3. "매주 보고서 쓰는 게 귀찮아요" 같은 일상의 불편함을 말하면, 클로드가 질문을 통해 요구사항을 정리하고, 30분 안에 첫 미니 프로젝트를 함께 만든다.

### 시나리오 2: 미니 프로젝트를 몇 개 완료한 사람

미니 프로젝트를 2~3개 마치면 클로드의 안내 수준이 바뀐다(Phase 전환). 점차 스스로 프로젝트를 제안하고, 복잡한 자동화를 설계하는 풀 프로젝트로 넘어간다.

---

## Phase 시스템 -- 교육 수준이 사용자에 맞춰 변한다

클로드 코드가 사용자의 경험 수준에 따라 안내 방식을 조절한다. CLAUDE.md에 정의된 규칙에 따라 자동으로 판단한다.

| Phase | 역할 | 설명 |
|---|---|---|
| Phase 1: 교사 | 모든 것을 설명 | 기본 개념부터 단계마다 멈추고 설명한다. 처음 시작하는 사람의 기본값. |
| Phase 2: 코치 | 핵심만 안내 | 사용자가 의사결정을 주도하고, 클로드는 필요한 부분만 짚어준다. |
| Phase 3: 협업자 | 사용자 주도 | 일반 클로드 코드 사용에 가까움. 사용자가 설계하고, 클로드가 실행한다. |

---

## 미니 프로젝트에서 풀 프로젝트로

```
미니 프로젝트 (Phase 1~2)          풀 프로젝트 (Phase 2~3)
------------------------------     --------------------------------
- 이 플레이북 안에서 진행            - 별도 프로젝트 폴더로 독립
- 기존 CLAUDE.md를 그대로 사용       - 전용 CLAUDE.md, spec.md 직접 작성
- 단순하고 빠른 자동화 연습          - 복합 자동화, 하네스(harness) 설계
- 결과물: mini-projects/ 에 저장     - 결과물: ~/projects/새_프로젝트/ 에 저장
```

- 미니 프로젝트: 플레이북 내에서 진행하는 작은 자동화. 반복 업무 하나를 자동화하는 수준.
- 풀 프로젝트: 독립 프로젝트로 분리하여 진행. CLAUDE.md(AI에게 맥락을 주는 설정 파일), spec.md(입출력 명세), planning.md(실행 계획) 등 "하네스(AI가 실행할 환경)"를 직접 설계한다.

---

## 기술 스택

| 기술 | 역할 | 설명 |
|---|---|---|
| Claude Code | 핵심 도구 | 터미널에서 대화로 작업을 시키는 AI CLI. 파일 생성, 코드 작성, 자동화 실행을 한다. |
| CLAUDE.md | 맥락 설정 | 프로젝트 루트에 놓는 텍스트 파일. AI가 "이 프로젝트에서 어떻게 행동해야 하는지" 읽는 설정서. |
| Git | 버전 관리 | 파일 변경 이력을 기록하는 도구. 실수해도 이전 상태로 되돌릴 수 있다. |
| Warp | 터미널 | 기본 터미널보다 보기 편한 최신 터미널 앱. 초보자에게 권장. |
| GitHub | 코드 저장소 | 프로젝트 파일을 온라인에 저장하는 서비스. |
| Markdown | 문서 형식 | 간단한 기호로 서식을 표현하는 텍스트 형식. README, 진행 기록 등 모든 문서에 사용. |

## 폴더 구조

```
claude-code-playbook/
  CLAUDE.md                    -- AI 행동 규칙 + 교육 모드 설정 (이 프로젝트의 핵심)
  README.md                    -- 프로젝트 소개 (지금 읽고 있는 파일)
  .gitignore                   -- Git이 추적하지 않을 파일 목록
  mini-projects/               -- 미니 프로젝트 결과물 (사용자 하위 폴더)
    {이름}/{프로젝트명}/        -- 각 미니 프로젝트의 코드, 기획서 등
  progress/                    -- 학습 진행 기록
    {이름}/                    -- 개인 진행 기록 파일
  docs/                        -- 문서 모음
    guides/                    -- 가이드 문서
      교육-레퍼런스.md          -- 교육 비유, 넛지 대사, 첫 세션 가이드
      phase1-2-교육모드.md      -- Phase 1-2 교육 모드 상세
      phase3-플러그인-가이드.md -- Phase 3 전환 시 소개하는 고급 기능 안내
    templates/                 -- 문서 템플릿
      progress-template.md     -- 진행 기록 양식
      case-study-template.md   -- 케이스 스터디 양식
      full-project-claude-md-template.md  -- 풀 프로젝트 CLAUDE.md 양식
      spec-template.md         -- 풀 프로젝트 스펙 양식
    case-studies/              -- 완료된 미니 프로젝트의 케이스 스터디
```

## 시작하기

### 필요한 것

- **Claude 구독** (Pro 또는 Max) -- 이것만 있으면 된다. 이미 Claude를 유료로 쓰고 있다면 추가 비용 없이 바로 시작할 수 있다. API 키는 필요 없다.

---

### 1단계: Warp 설치하기

Warp는 최신 터미널 앱이다. 기본 터미널보다 보기 편하고 쓰기 쉬워서, 처음 시작하는 분에게 추천한다.

1. [warp.dev/download](https://www.warp.dev/download) 에 접속한다.
2. 본인 컴퓨터에 맞는 버전을 다운로드한다 (Mac / Windows).
3. 다운로드된 파일을 실행해서 설치한다.
   - **Mac**: `.dmg` 파일 열기 → Warp를 Applications 폴더로 드래그
   - **Windows**: `.exe` 파일 실행 → 설치 완료까지 대기
4. 설치가 끝나면 Warp를 실행한다.
   - 처음 실행 시 회원가입 화면이 나올 수 있다. Google 계정 등으로 가입하면 된다.

실행하면 창이 하나 뜬다. 여기에 글자를 입력하면 컴퓨터가 그대로 실행하는 창이다. 그냥 텍스트 대화창이라고 생각하면 된다.

<details>
<summary>Warp 없이 기본 터미널로 하고 싶다면 (클릭해서 펼치기)</summary>

**Mac의 경우:**
`Cmd + 스페이스바` → "터미널" 검색 → 열기

**Windows의 경우:**
`Win 키` → "PowerShell" 검색 → **"Windows PowerShell"** 열기
> "명령 프롬프트"나 "CMD"가 아니다. 반드시 **PowerShell**을 열어야 한다.

</details>

---

### 2단계: Git 설치

Warp(또는 터미널)에서 아래를 복사해서 붙여넣고 `Enter`:

**Mac의 경우:**
```
xcode-select --install
```
- 팝업 창이 뜬다. **"설치"** 를 클릭한다.
- 설치에 몇 분 걸릴 수 있다. 끝날 때까지 기다린다.
- "이미 설치되어 있습니다" 같은 메시지가 나오면 이미 된 것이다. 다음 단계로 넘어간다.

**Windows의 경우:**
```
winget install Git.Git
```
- 영어로 설치 과정이 나온다. 끝날 때까지 기다린다.
- 설치가 끝나면 **Warp를 닫았다가 다시 열어야 한다**.

---

### 3단계: 클로드 코드 설치

Warp(또는 터미널)에서 아래를 복사해서 붙여넣고 `Enter`:

**Mac의 경우:**
```
curl -fsSL https://claude.ai/install.sh | bash
```

**Windows의 경우:**
```
irm https://claude.ai/install.ps1 | iex
```

- 영어로 뭔가 주르륵 나온다. 정상이다. 설치가 진행되는 것이다.
- 글자가 멈추고 더 이상 아무 일도 안 일어나면 설치가 끝난 것이다.
- 이제 **Warp를 닫았다가 다시 열어야 한다**.

> 왜 닫았다 다시 여느냐면: 방금 설치한 것들을 컴퓨터가 인식하려면 Warp를 새로 열어야 한다. 한 번만 하면 된다.

---

### 4단계: 플레이북 다운로드 + 클로드 코드 실행

마지막 단계다. 아래를 통째로 복사해서 붙여넣고 `Enter`:

**Mac의 경우:**
```
git clone https://github.com/vorovong/claude-code-playbook.git ~/projects/claude-code-playbook && cd ~/projects/claude-code-playbook && claude
```

**Windows의 경우:**
```
git clone https://github.com/vorovong/claude-code-playbook.git "$HOME\projects\claude-code-playbook"; cd "$HOME\projects\claude-code-playbook"; claude
```

- 파일이 다운로드되고, 자동으로 클로드 코드가 시작된다.

---

### 5단계: 처음 실행 -- 로그인하기 (최초 1회만)

처음 실행하면 몇 가지 질문이 나온다. 하나씩 따라가면 된다.

**5-1. 텍스트 스타일 선택**

터미널에 글자가 어떻게 보일지 고르는 화면이 나온다.
화살표 키(`위/아래`)로 골라서 `Enter`. 아무거나 골라도 된다.

**5-2. 로그인 방법 선택**

로그인 방법을 묻는 화면이 나온다.
**"Claude account with subscription"** 을 선택한다 (화살표로 이동 후 `Enter`).

> "Anthropic Console"이 아니다. 구독자는 위쪽 옵션을 선택하면 된다.

**5-3. 브라우저에서 로그인**

`Enter`를 누르면 웹 브라우저가 자동으로 열린다.
평소 Claude 웹사이트(claude.ai)에 로그인하는 것과 똑같이 로그인한다.

로그인이 완료되면 브라우저에 "성공" 메시지가 뜬다.
이제 터미널로 돌아간다.

**5-4. 보안 안내 + 폴더 신뢰 확인**

터미널에 영어로 보안 관련 안내가 나온다. `Enter`를 누른다.
그다음 "이 폴더를 신뢰하시나요?"라는 질문이 나온다.
**"Yes, proceed"** 를 선택하고 `Enter`.

---

### 6단계: 완료

`>` 표시가 나오면 설치 끝, 대화 준비 완료다.

클로드가 먼저 말을 건다. "무엇을 자동화하고 싶으세요?" 같은 질문부터 시작해서, 30분 안에 하나의 결과물을 같이 만들게 된다.

어렵게 느껴지면 언제든 "잘 모르겠어요"라고 말해도 된다. 클로드가 다시 쉽게 설명해준다.

---

## 내 플레이북 커스텀하기

이 플레이북은 **출발점**이지 완성품이 아니다. 프로젝트를 진행하면서 자연스럽게 자기만의 것으로 바꿔나가면 된다.

- Phase 2부터: CLAUDE.md를 직접 수정해서 클로드의 행동을 바꿔볼 수 있다.
- Phase 3부터: 풀 프로젝트에서 자기만의 CLAUDE.md, 스킬, 에이전트를 설계한다.
- 커스텀하는 과정 자체가 학습이다. 처음부터 완벽할 필요 없다.
