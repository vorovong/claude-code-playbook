# AI 시대의 코드 판단력 — 커리큘럼 리서치 계획서

## 연구 질문

> AI가 코드를 대신 써주는 시대에, 그 코드의 옳고 그름을 판단하는 선구안은 어떻게 기르는가?

세부 질문:
1. 세계 탑티어 대학과 기관은 이 문제를 어떻게 교육하고 있는가?
2. 그 중 비전공자가 독학(2인 페어)으로 따라갈 수 있는 것은 무엇인가?
3. 이미 AI와 실전 프로젝트를 운영 중인 학습자에게 맞는 경로는 무엇인가?

## 배경

### 문제 정의

AI Native Engineer 아티클의 프레임:
- **도구 지식** (언어 문법, API 사용법) → AI가 대체 중
- **원리 지식** (자료구조, 시스템, 알고리즘) → AI 코드를 판단하는 기초. 더 중요해짐

DORA 2025 실증: PR 생성량 98% 증가 vs 소프트웨어 성과 무변화.
→ 코드를 만드는 능력이 아니라, 만들어진 코드를 판단하는 능력이 병목.

### 학습자: 김민호

| 항목 | 현재 |
|---|---|
| 코드 읽기 | Python 단어 수준 유추. 라이브러리/프레임워크 코드는 판단 어려움 |
| 원리 지식 | if/else, function, OOP 개념 수준. 자료구조/알고리즘 기초 부족 |
| AI 도구 | Claude Code 2개월, Gemini CLI, 3대 LLM Pro 구독 |
| 실전 | 도란(Doran) — Next.js/PostgreSQL/Gemini/Docker SaaS. 30세션, 134건 의사결정, E2E 62/62 pass |
| 투자 | 풀타임, 주 20시간 |

**핵심 갭**: 실전은 앞서가는데 원리가 따라오지 못함. AI가 만든 코드가 "돌아가니까 OK"인지 "구조적으로 맞는지" 구분할 눈이 없음.

**기르고 싶은 선구안**:
- 코드를 빠르게 읽고 구조를 파악하는 속도
- 논리적 모순과 엣지 케이스를 잡아내는 눈
- 트레이드오프를 근거와 함께 판단하는 능력
- AI가 생성한 코드/로직/설계를 비판적으로 평가하는 힘

## 조사 항목

각 커리큘럼(#1~#6)에 대해:

| 항목 | 질문 |
|---|---|
| 구조 | 몇 주, 몇 단계? 순서는? |
| 전제 조건 | 어떤 수준의 학습자를 대상으로 하는가? |
| 핵심 주제 | 무엇을 가르치는가? (주차별 토픽) |
| 코드 판단력 훈련 | 코드 리뷰, 디버깅, 설계 비평, 트레이드오프 분석 등 "읽고 판단하는" 훈련이 있는가? |
| 원리 vs 도구 비중 | CS 원리와 특정 도구/언어 학습의 비율은? |
| AI 정책 | AI를 금지/허용/필수 중 어떻게 다루는가? |
| 공개 자료 | 실라버스, 과제, 영상 등 독학 가능한 자료가 있는가? |

참고 자료(#7~#12)에서는:
- 커리큘럼 설계 의도, 업계 트렌드, 교육 철학
- "코드 판단력"을 기르는 방법에 대한 추가 관점

## 평가 프레임

| 기준 | 질문 | 가중치 |
|---|---|---|
| **코드 판단력** | 모순/엣지케이스/트레이드오프를 식별하는 훈련이 있는가? | 높음 |
| **원리 지식** | 자료구조, 시스템, 알고리즘 등 원리를 다루는가? | 높음 |
| **AI 네이티브** | AI 생성 코드를 비판적으로 평가하는 역량을 키우는가? | 높음 |
| **접근성** | 비전공자(코드 읽기 수준)가 독학으로 따라갈 수 있는가? | 중간 |
| **실용성** | 실제 프로젝트(도란 등)에 바로 적용 가능한가? | 중간 |
| **자료 공개** | 실라버스/과제가 공개되어 있는가? | 중간 |
| **커버리지** | CS 원리 → AI 활용 개발 → 1인 SaaS 운영 경로가 있는가? | 낮음 |
| **2인 학습** | 페어/소규모 학습에 적합한 구조인가? | 낮음 |

가중치 근거: 핵심 목표가 "선구안"이므로, 판단력/원리/AI 비판이 최우선. 접근성과 실용성은 필터 조건. 커버리지와 2인 학습은 커스터마이징으로 해결 가능.
