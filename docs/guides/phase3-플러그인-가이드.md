# 클로드 코드 플러그인 가이드

> **이 문서는 클로드 코드가 사용자에게 적절한 시점에 전달하기 위한 가이드입니다.**
>
> - **전달 시점**: Phase 3를 마치고, 사용자가 풀 프로젝트로 전환하려는 순간
> - **전달 방식**: 아래 "서프라이즈 멘트"를 자연스럽게 전달한 후, 단계별로 안내
> - **핵심 원칙**: 기본기를 충분히 체득한 상태에서 도구를 만나야, 그 가치를 진정으로 이해할 수 있다

---

## 서프라이즈 멘트

> 클로드 코드가 사용자에게 전달할 멘트 (상황에 맞게 자연스럽게 변형):

"잠깐, 풀 프로젝트로 넘어가기 전에 알려줄 게 있어.

사실 클로드 코드에는 **플러그인**이라는 확장 기능이 있어. 지금까지 일부러 안 알려준 거야.

지금까지는 클로드 코드의 핵심 원리를 직접 체득하고, 혼자 하는 것의 장점과 한계를 몸으로 익히는 게 먼저였거든. CLAUDE.md 쓰는 법, 플랜 모드, 서브에이전트... 이런 기본기를 충분히 익힌 상태에서 플러그인을 만나야 '아, 이게 있었으면 그때 편했겠다!'라는 깨달음이 생기니까.

이제 네가 혼자 프로젝트를 진행하려 하잖아. 그래서 알려줄 때가 된 거야. 꼭 필요한 부가기능들이니까, 공부하고 사용해봐."

---

## 플러그인이 뭔지

스마트폰에 앱을 설치하듯이, 클로드 코드에도 기능을 추가할 수 있어.

| 비유 | 실제 |
|---|---|
| 스마트폰 | 클로드 코드 (기본 기능) |
| 앱 | 플러그인 (골라서 설치하는 추가 기능) |
| 앱스토어 | 마켓플레이스 (플러그인이 모여 있는 곳) |

설치는 두 줄이면 끝나:

```
/plugin marketplace add [마켓플레이스 주소]   ← 앱스토어 등록
/plugin install [플러그인 이름]               ← 앱 설치
```

## 단계별 도입

한꺼번에 다 설치하지 말고, 필요할 때 하나씩.

### 1단계: 보호망 설치 (풀 프로젝트 시작할 때)

풀 프로젝트는 미니 프로젝트보다 복잡하고, 실수의 영향이 더 크다. 먼저 "보호망"부터 설치한다.

#### Context7 — 최신 문서 자동 주입

- **뭘 하는 건지**: 라이브러리 공식 문서를 실시간으로 클로드 코드에게 전달
- **왜 필요한지**: 클로드 코드가 오래된 정보로 잘못된 코드를 쓰는 걸 방지
- **비유**: 요리사에게 최신 레시피북을 항상 옆에 두게 하는 것

#### Security Guidance — 보안 자동 감시

- **뭘 하는 건지**: 코드를 수정할 때 보안 취약점을 자동으로 스캔하고 차단
- **왜 필요한지**: 비개발자는 보안 문제를 눈으로 볼 수 없으니, 자동 감시가 필수
- **비유**: 집을 리모델링할 때 안전 검사관이 옆에서 체크하는 것

---

### 2단계: 체계적 개발 (프로젝트가 커질 때)

코드가 많아지고 구조가 복잡해지면, 체계적인 프로세스가 필요하다.

#### Superpowers — 시니어 개발자 방법론

- **뭘 하는 건지**: 코드를 바로 쓰지 않고, brainstorm → plan → implement 순서를 강제
- **왜 필요한지**: 클로드 코드가 "시니어 개발자"처럼 체계적으로 일하게 만듦
- **비유**: 건축가가 설계도 없이 집을 짓지 않듯이, 코드도 설계부터 하게 하는 것
- **참고**: GitHub 스타 40.9k — 가장 인기 있는 Claude Code 플러그인

```
/plugin marketplace add obra/superpowers-marketplace
/plugin install superpowers@superpowers-marketplace
```

설치하면, "이런 거 만들어줘"라고 말하기만 하면 클로드 코드가 알아서 질문부터 시작하고, 설계를 보여주고, 승인하면 구현으로 넘어가는 흐름이 자동으로 작동한다. 별도 명령어를 외울 필요가 없다.

#### Code Review — 자동 코드 리뷰

- **뭘 하는 건지**: AI가 코드를 자동 리뷰하고, 문제점을 심각도별로 보고
- **왜 필요한지**: 혼자 개발할 때 놓치는 실수를 잡아줌
- **비유**: 혼자 쓴 보고서를 제출 전에 동료에게 검토받는 것

---

### 3단계: 자율 실행 (충분히 익숙해진 후)

프로젝트 경험이 쌓이면, 클로드 코드가 밤새 혼자 일하게 할 수 있다.

**주의: 반드시 1~2단계를 먼저 익힌 후에 사용할 것. 자율 실행은 검증 없이 돌리면 위험하다.**

#### Ralph Loop — 자율 반복 실행 (Anthropic 공식)

- **뭘 하는 건지**: 클로드 코드가 스스로 반복하면서 작업을 완료할 때까지 멈추지 않음
- **왜 필요한지**: 큰 작업을 밤새 돌려놓고, 아침에 결과를 확인할 수 있음
- **비유**: 식기세척기에 그릇 넣고 버튼 누르는 것. 끝나면 알아서 멈춤.

```
/plugin marketplace add anthropics/claude-code
/plugin install ralph-wiggum@claude-plugins-official
/ralph-loop "작업 설명" --max-iterations 20
```

#### Flow-Next — 통합 오케스트레이션 (최종 무기)

- **뭘 하는 건지**: Superpowers(설계) + Ralph(자율 실행)를 하나로 합치고, 자동 리뷰까지 추가
- **왜 필요한지**: 자율 실행 중에도 품질 검증이 자동으로 이루어짐. 원래 목표에서 벗어나는 것도 자동 방지.
- **비유**: 식기세척기가 그릇이 깨끗한지까지 확인하고, 안 되면 다시 돌리는 것

```
/plugin marketplace add https://github.com/gmickel/gmickel-claude-marketplace
/plugin install flow-next
/flow-next:setup
```

Flow-Next는 개발 프로세스 관리 경험이 어느 정도 쌓인 후에 도입하는 게 좋다. epic, task, dependency graph 같은 개념을 이해해야 제대로 활용할 수 있기 때문이다.
