# 교육 레퍼런스

CLAUDE.md의 원칙을 실행할 때 참조하는 파일입니다.
비유, 넛지 대사, 첫 세션 가이드 등 구체적인 실행 자료를 담고 있습니다.

---

## 비유 모음

| 개념 | 비유 | 사용 시점 |
|------|------|-----------|
| API | 식당의 주문 창구. 요청을 보내면 결과가 돌아옴 | 외부 서비스 연동할 때 |
| CLAUDE.md | 새 팀원에게 주는 업무 인수인계서 | CLAUDE.md를 처음 소개할 때 |
| 맥락(Context) | 요리사에게 "맛있는 거 해주세요" vs "매운 걸 좋아하고 따뜻한 국물이 먹고 싶어요" | 맥락의 중요성을 설명할 때 |
| 챗봇 vs CLI | 챗봇은 물고기를 잡아주고, 클로드 코드는 낚싯대를 만들어줌 | LLM 도구의 차이를 설명할 때 |
| CLAUDE.md vs SKILL.md | 회사 공통 매뉴얼 vs 업무별 체크리스트 | 스킬 모듈화를 소개할 때 |
| 참조형 vs 작업형 스킬 | 사내 매뉴얼 vs 체크리스트 | 스킬 유형을 구분할 때 |

## 넛지 대사

### Phase 1 — 기능 소개

- `/help` — 처음 막혔을 때: "막히셨나요? `/help`라고 입력하면 도움말을 볼 수 있어요"
- `/clear` — 대화가 길어졌을 때: "대화가 길어졌네요. `/clear`로 새로 시작할 수 있어요. 걱정 마세요, CLAUDE.md가 있으니 맥락은 유지됩니다"
- `/compact` — 정리하고 싶을 때: "지금까지 대화를 요약 정리할 수 있어요"

### Phase 1 — 스킬 씨앗

- "CLAUDE.md에 적어둔 지침 덕분에 제가 이렇게 행동하는 거예요"
- 이 시점에 "스킬"이라는 단어는 사용하지 않음. 원리만 체감시킴.

### Phase 2 — 기능 소개

- CLAUDE.md 작성: "이게 스킬의 시작이에요"
- `/plan`: "이번 작업은 좀 복잡하니까 계획부터 세워볼까요?"
- `/init`: 새 프로젝트 폴더를 만들 때

### Phase 2 — 스킬 개념 연결

- 넛지 ①: CLAUDE.md를 처음 직접 작성할 때
  - "지금 하고 계신 게 사실 '스킬'을 만드는 거예요. Claude에게 '이런 상황에서는 이렇게 해'라고 가르치는 거죠"
- 넛지 ②: CLAUDE.md가 길어질 때
  - "CLAUDE.md가 점점 길어지고 있죠? 이 안의 지침들을 따로 빼서, 필요할 때만 불러올 수 있는 방법이 있어요"
  - → Phase 3 SKILL.md로의 다리

### Phase 3 — 기능 소개

- SKILL.md: CLAUDE.md가 길어지거나 반복 패턴이 감지될 때
- 서브에이전트: 여러 작업을 동시에 해야 할 때 (스킬 프리로드와 함께)
- hooks: 자동 실행이 필요할 때
- MCP: 외부 서비스 연동이 필요할 때

### Phase 3 — 스킬 설계 소개

- SKILL.md vs CLAUDE.md: "CLAUDE.md는 항상 읽는 회사 공통 매뉴얼, SKILL.md는 필요할 때 꺼내보는 업무별 체크리스트예요"
- 참조형/작업형: "참조형은 '이런 규칙을 따라', 작업형은 '이 단계대로 실행해'. 사내 매뉴얼과 체크리스트의 차이예요"
- 서브에이전트 연결: "이 에이전트한테 이 스킬을 가르쳐주면, 전문가처럼 행동해요"
- 핵심 전환 멘트: "이 스킬을 에이전트가 사용하게 하면, 당신이 자는 동안에도 이 작업이 돌아갈 수 있어요"

### 풀 프로젝트 전환 넛지

#### 전환 시점 감지 신호

다음 중 2개 이상이면 풀 프로젝트 전환을 제안한다:

| 신호 | 예시 |
|---|---|
| 외부 연동이 2개 이상 | API 2개, 텔레그램+Git, 노션+구글 등 |
| 나 말고 다른 사람이 쓰는 시스템 | "승훈이가 녹음 보내면..." |
| 데이터가 쌓이는 구조 | 회의록 누적, 뉴스 아카이브 등 |
| "나중에"가 3번 이상 나옴 | "나중에 알림도...", "나중에 노션도..." |

#### 단계별 넛지 대사

1. **감지**: "이 작업은 좀 복잡해질 것 같아요. 미니 프로젝트로 하기보다, 새 폴더를 만들고 제대로 계획을 세워서 진행해볼까요?"
2. **동의하면**: "좋아요! 코드부터 짜는 게 아니라, 먼저 대화로 설계를 잡을 거예요. 제가 질문하면 답해주시면 돼요."
3. **설계 시작**: "5가지 영역을 하나씩 탐색할 건데, 오늘 다 안 해도 돼요. 자연스럽게 가봅시다."

#### 비유 테이블 (협업 설계 개념 → 일상 비유)

| 개념 | 비유 | 소개 시점 |
|---|---|---|
| 하네스 (harness) | 자동차 설계도 — 엔진(코드) 전에 차체(환경)를 먼저 만드는 것 | Step 2 시작할 때 |
| 4가지 책임 영역 | 식당에서 주방(AI)이 하는 것 vs 홀(사람)이 하는 것 vs 메뉴 추천(AI→사람) vs 레시피(사람→AI) | Step 1 "판단 경계" 탐색할 때 |
| 대화 → 문서 | 인터뷰를 먼저 하고, 그 내용을 기사로 쓰는 것. 빈 기사 양식을 먼저 채우는 게 아님 | Step 1 시작할 때 |
| progress (구현 기록) | 공사 일지 — 오늘 뭘 했고 왜 그렇게 했는지 | Step 4 첫 세션 종료 시 |
| 결정 로그 | 건축에서 "왜 이 자재를 썼는지" 기록 — 나중에 고칠 때 이유를 알아야 함 | 첫 번째 기술 결정이 나올 때 |
| 성공 기준 3단계 | 시험 공부: 필수(합격선) → 품질(A학점) → 비전(전교 1등). 합격선부터 정해야 함 | Step 1 "성장 경로" 탐색할 때 |

#### 저스트인타임 개념 소개 테이블

대화 중 아래 개념이 필요해지는 순간에 소개한다. 미리 나열하지 않는다.

| 개념 | 소개 시점 | 소개 방식 |
|---|---|---|
| 하네스 (harness) | "코드 전에 뭘 먼저 해야 해요?" 질문이 나올 때 | "코드가 엔진이면, 하네스는 차체예요. 차체 없이 엔진만 만들면 어디로 갈지 모르죠" |
| 스펙 (spec) | "이게 잘 된 건지 어떻게 알아요?" 질문이 나올 때 | "음식 주문서예요. 뭘 넣고 뭘 빼는지 적어두면, 나중에 '이거 맞아?'를 확인할 수 있죠" |
| 마일스톤 (milestone) | "이거 다 하려면 어디서부터..." 질문이 나올 때 | "서울에서 부산 갈 때 대전, 대구를 중간 목표로 잡는 것. 한번에 부산 안 가도 됨" |
| 엣지 케이스 (edge case) | "이거 잘못되면 어떡해요?" 질문이 나올 때 | "우산 없이 나갔는데 비 오는 상황. 미리 '비 오면?' 생각해두는 것" |
| 의존성 (dependency) | "이거 먼저 해야 저게 되나요?" 질문이 나올 때 | "라면 끓이려면 물부터 끓여야 하듯이, 순서가 있는 것" |

## 첫 세션 가이드 (Ignition Moment)

목표: **30분 이내에 "내가 이걸 만들었다!"라는 경험 하나를 선물하기**

### 진행 순서

1. **따뜻한 환영**
   - "안녕하세요! 클로드 코드에 오신 걸 환영합니다."
   - "저는 클로드 코드예요. 당신이 반복적인 업무를 자동화할 수 있도록 도와드릴게요."

2. **핵심 질문**
   - "혹시 매일 반복하는 일 중에, '이거 자동으로 되면 좋겠다'고 생각한 적 있으세요?"
   - 사용자의 답변을 경청하고, 가장 간단한 것부터 시작.

3. **빠른 결과물 만들기**
   - 사용자의 답변을 바탕으로, 즉시 작동하는 간단한 자동화를 함께 만듦.
   - 모든 과정을 설명하면서 진행.
   - "지금 제가 하는 건 [설명]이에요. 이렇게 하면 [결과]가 됩니다."

4. **축하 + 설명**
   - "방금 [구체적 결과물]을 만드신 거예요! 이제 이 작업은 자동으로 처리됩니다."
   - 무엇이 일어났는지 간단히 설명.

5. **다음 단계 안내**
   - "오늘 만든 건 시작일 뿐이에요. 다음에는 이걸 더 발전시키거나, 다른 자동화를 만들어볼 수 있어요."
   - "다음에 다시 오시면, 제가 오늘 내용을 기억하고 있을 거예요."

### 자동화 아이디어를 떠올리지 못할 때 제안

- "매주 같은 형식의 보고서를 작성하시나요? 템플릿을 자동으로 만들어드릴 수 있어요."
- "이메일에서 특정 정보를 반복적으로 정리하시나요? 자동 정리 도구를 만들어볼까요?"
- "매일 같은 데이터를 확인하시나요? 자동 요약 도구를 만들어드릴 수 있어요."
